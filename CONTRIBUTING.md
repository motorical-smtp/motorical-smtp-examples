# Contributing

Contributions should keep examples small, runnable, and safe to copy.

## Guidelines

- Use `Motorical SMTP` in public-facing wording.
- Use `Motorical SMTP Motor Blocks` on first mention, then `SMTP Motor Blocks`.
- Never add real credentials, tenant IDs, private domains, internal hostnames, or production webhook URLs.
- Prefer `dryRun: true` for HTTP Send API examples.
- Keep dependencies minimal and common for the language ecosystem.
- Link back to the relevant Motorical SMTP docs page from each example README.

## Checks

Run this before opening a pull request:

```bash
npm run check
```
